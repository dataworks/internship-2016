Purpose

Data Works has chosen the five of us as interns to come in and try and improve a real world process. This started as a way to help Data Works go through the hundreds of resumes that they receive every year for the internship program, but we hope that we have created something that can do a little more than that. The process of going through all the resumes that were submitted to any company is a long and tedious process, and sometimes in the end there are one or two amazing candidates. Well we have come up with a way to cut this time down to almost nothing!

Overview

Over the period of nine weeks, we have come up with an application that will make going through resumes and finding the best applicants as simple as clicking a few buttons. SmartHire will take in all of the resumes that a company receives over time, read through all of them, and assign each applicant a score. This score is based on a machine learning algorithm that builds a profile of an ideal candidate based on the user's selections. Using this score, the user can easily get an idea of whether or not the individual would be a good fit for the company. Along with the score, our application will go through the resume and pull out the applicant's name, school, GPA, degree, contact information, skills, links, and current position to help give a brief overview of the person.

Features

To be more user-friendly, we have also created some helpful options that will enhance the process. There are different views that the user can look at, starting off with the new view. The other views include favorites, review, and archive. In order to put individual applicants into each view, there are different buttons under the options column. There are six different options in general: favorite, review, archive, view resume, expand, and send back. The first three options that are displayed deal with the different applicant views. Once clicked, the applicant will be sent from the table to whichever view was selected. The send back button will send an applicant back to the new view. If the user likes an applicant or wants to learn more about him or her, the user will have two options: expand or view resume. The expand option will expand the row on the table for the given applicant, and it will show all of the applicant's skills, more about his or her education, and any links that were provided. Another option is to click view resume which is connected to the applicant's full resume that was submitted.

Technology

These past nine weeks have allowed us to learn so much, and we have all come so far in our understandings of the different types of technologies that are used in the working world. We have used Spack, ElastiSearch, Apache Tika, Apache OpenNLP, MLlib, AWS S3/HDFS, Github API, LinkedIn API, GoogleMaps API, Node.js, Express, Bootstrap, Chart.js, Mocha and Chai for testing, and AngularJS. To help with the tesing and deploying the application, we used Jenkins. Github was used to share/link all of our files, which allowed us to all work together.
